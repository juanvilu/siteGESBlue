¿Cómo agregar un campo personalizado con soporte de catálogos?

 

Autor: Margarita Lozano Neveros

Conoceremos dicho proceso mediante el siguiente ejemplo.

Vamos a definir un campo personalizado para alumnos llamado NIVEL_INGLES, de tipo Alfanumérico con longitud máxima de 10 caracteres. Los valores que requeriremos introducir son NINGUNO, BASICO, MEDIO, BUENO o PERFECTO.

Para evitar que el usuario ingrese otros valores, crearemos una tabla dentro de la misma base de datos de Interbase con la siguiente estructura:

CREATE TABLE CAT_NIVEL_INGLES (
ID_ESCUELA SMALLINT NOT NULL,
NIVEL_INGLES VARCHAR(10) NOT NULL,
DESCRIPCION VARCHAR(100) )



Y la llenaremos con los siguientes valores:

INSERT INTO cat_nivel_ingles (ID_ESCUELA, NIVEL_INGLES, DESCRIPCION )
VALUES (1, “NINGUNO”, “Sin conocimientos de Inglés” )

INSERT INTO cat_nivel_ingles (ID_ESCUELA, NIVEL_INGLES, DESCRIPCION )
VALUES (1, “BASICO”, “Conocimientos básicos de Inglés” )

INSERT INTO cat_nivel_ingles (ID_ESCUELA, NIVEL_INGLES, DESCRIPCION )
VALUES (1, “MEDIO”, “Conocimientos aceptables de Inglés” )

INSERT INTO cat_nivel_ingles (ID_ESCUELA, NIVEL_INGLES, DESCRIPCION )
VALUES (1, “BUENO”, “Se expresa con fluidez en Inglés” )

INSERT INTO cat_nivel_ingles (ID_ESCUELA, NIVEL_INGLES, DESCRIPCION )
VALUES (1, “PERFECTO”, “Se expresa y entiende a la perfección el Inglés” )

Observe que en la opción Tabla de Catálogo de referencia, podemos seleccionar la tabla recien creada, además de que la Descripción o nombre del campo debe ser extactamente igual al nombre de la Tabla anteponiendole el prefijo CAT_ .

Después de guardar los cambios, se sugiere salir del sistema para reiniciar el buffer de campos adicionales.

Ahora en la ficha del alumno, nuestro campo personalizado será llenado con una lista que permitirá utilizar los valores de la tabla recién creada y por lo tanto, impedirá incorporar valores incorrectos.
